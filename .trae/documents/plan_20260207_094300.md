## 创建 audio-transcriber-zipformer 项目

### 任务概述
基于 `audio-transcriber-tiny` 项目结构创建 `audio-transcriber-zipformer` 项目，使用 Zipformer Transducer 模型进行语音识别。

### 实施步骤

#### 1. 创建项目目录结构
在 `audio-transcriber-all` 文件夹下创建 `audio-transcriber-zipformer` 目录：
- `src/` - 存放源代码
- `files/` - 存放转录结果

#### 2. 复制并修改项目文件
从 `audio-transcriber-tiny` 复制所有文件到 `audio-transcriber-zipformer`：
- `src/index.ts`
- `src/transcriber.ts`
- `src/sherpa-onnx-node.d.ts`
- `package.json`
- `tsconfig.json`

#### 3. 修改 package.json
更新 `audio-transcriber-zipformer/package.json`：
- name: `audio-transcriber-zipformer`
- description: `Audio transcription module using VAD v4 + Zipformer Transducer`

#### 4. 更新 TypeScript 类型定义
修改 `src/sherpa-onnx-node.d.ts`，添加 Transducer 模型配置接口：
```typescript
export interface OfflineTransducerModelConfig {
  encoder: string;
  decoder: string;
  joiner: string;
}

export interface OfflineModelConfig {
  transducer?: OfflineTransducerModelConfig;
  tokens: string;
  numThreads: number;
  provider: string;
  debug: number;
}
```

#### 5. 修改 transcriber.ts
更新 `src/transcriber.ts` 中的模型创建逻辑：
- 添加 Transducer 配置接口
- 修改 `createRecognizer()` 方法，使用 Transducer 配置结构
- 更新日志输出，显示 "Processing audio with VAD v4 + Zipformer..."

#### 6. 修改 test.js
更新 `test.js` 中的模型配置：
```javascript
const config = {
  transducer: {
```

#### 7. 构建项目
运行 `npm run build` 编译 TypeScript 代码

### Zipformer 模型特点

1. **Transducer 架构** - 使用 encoder + decoder + joiner 结构
2. **int8 量化** - 模型文件都是 int8 量化版本
3. **高性能** - Zipformer 是一个高性能的语音识别模型
4. **多语言支持** - 支持双语和多语言

### 预期优势

使用 Zipformer Transducer 模型后，预期会有以下优势：
- **更高的准确率** - Zipformer 是一个高性能的语音识别模型
- **int8 量化** - 更小的模型文件，更低的内存占用，更快的推理速度
- **VAD v4 集成** - 更精确的语音边界检测
- **Transducer 架构** - 可能比 Moonshine 和 Whisper 在某些场景下表现更好
