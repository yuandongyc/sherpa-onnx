## 创建 audio-transcriber-moonshine-tiny-int8 项目

### 任务概述
基于 `audio-transcriber-tiny` 项目创建 `audio-transcriber-moonshine-tiny-int8` 项目，使用 Whisper Tiny 模型的 int8 量化版本，以获得更好的性能和更低的内存占用。

### 实施步骤

#### 1. 创建项目目录结构
在 `audio-transcriber-all` 文件夹下创建 `audio-transcriber-moonshine-tiny-int8` 目录：
- `src/` - 存放源代码
- `files/` - 存放转录结果

#### 2. 复制项目文件
从 `audio-transcriber-tiny` 复制所有文件到 `audio-transcriber-moonshine-tiny-int8`：
- `src/index.ts`
- `src/transcriber.ts`
- `src/sherpa-onnx-node.d.ts`
- `package.json`
- `tsconfig.json`

#### 3. 修改 package.json
更新 `audio-transcriber-moonshine-tiny-int8/package.json`：
- name: `audio-transcriber-moonshine-tiny-int8`
- description: `Audio transcription module using VAD + Whisper (Tiny model int8 quantized)`

#### 4. 修改 test.js
更新 `audio-transcriber-moonshine-tiny-int8/test.js` 中的 Whisper 模型路径：
- encoder: `tiny.en-encoder.int8.onnx`（使用 int8 量化版本）
- decoder: `tiny.en-decoder.int8.onnx`（使用 int8 量化版本）
- tokens: `tiny.en-tokens.txt`
- VAD 模型保持使用 `silero_vad_v4.onnx`

#### 5. 构建项目
运行 `npm run build` 编译 TypeScript 代码

#### 6. 运行测试
执行 `node test.js` 测试 int8 量化版本的效果

### 文件结构
```
audio-transcriber-all/
└── audio-transcriber-moonshine-tiny-int8/
    ├── src/
    │   ├── index.ts
    │   ├── transcriber.ts
    │   └── sherpa-onnx-node.d.ts
    ├── files/           (用于存放转录结果)
    ├── package.json
    ├── tsconfig.json
    └── test.js
```

### 预期优势
使用 int8 量化版本后，预期会有以下改进：
- **更快的模型加载速度** - int8 模型文件更小
- **更低的内存占用** - 8 位整数表示节省内存
- **更快的推理速度** - 在支持 int8 的 CPU 上可以加速
- **几乎相同的准确率** - int8 量化对语音识别任务影响很小
