## 创建 audio-transcriber-moonshine-base-int8 项目

### 任务概述
基于 `audio-transcriber-moonshine-tiny-int8` 项目结构创建 `audio-transcriber-moonshine-base-int8` 项目，使用 Moonshine Base int8 模型进行语音识别。

### 实施步骤

#### 1. 创建项目目录结构
在 `audio-transcriber-all` 文件夹下创建 `audio-transcriber-moonshine-base-int8` 目录：
- `src/` - 存放源代码
- `files/` - 存放转录结果

#### 2. 复制并修改项目文件
从 `audio-transcriber-moonshine-tiny-int8` 复制所有文件到 `audio-transcriber-moonshine-base-int8`：
- `src/index.ts`
- `src/transcriber.ts`
- `src/sherpa-onnx-node.d.ts`
- `package.json`
- `tsconfig.json`

#### 3. 修改 package.json
更新 `audio-transcriber-moonshine-base-int8/package.json`：
- name: `audio-transcriber-moonshine-base-int8`
- description: `Audio transcription module using VAD v4 + Moonshine (Base model int8 quantized)`

#### 4. 更新 TypeScript 类型定义
修改 `src/sherpa-onnx-node.d.ts`，添加 Moonshine 模型配置接口：
```typescript
export interface OfflineMoonshineModelConfig {
  preprocessor: string;
  encoder: string;
  uncached_decoder: string;
  cached_decoder: string;
}

export interface OfflineModelConfig {
  moonshine?: OfflineMoonshineModelConfig;
  tokens: string;
  num_threads?: number;
  provider?: string;
  debug?: number;
}
```

#### 5. 修改 transcriber.ts
更新 `src/transcriber.ts` 中的模型创建逻辑：
- 添加 Moonshine 配置接口
- 修改 `createRecognizer()` 方法，使用 Moonshine 配置结构
- 更新日志输出，显示 "Processing audio with VAD v4 + Moonshine (Base int8)..."

#### 6. 修改 test.js
更新 `test.js` 中的模型配置：
```javascript
const config = {
  moonshine: {
    preprocessor: 'D:/MyProjects/c_projects/sherpa-onnx/all_models/sherpa-onnx-moonshine-base-en-int8/preprocess.onnx',
    encoder: 'D:/MyProjects/c_projects/sherpa-onnx/all_models/sherpa-onnx-moonshine-base-en-int8/encode.int8.onnx',

```

#### 7. 构建项目
运行 `npm run build` 编译 TypeScript 代码

#### 8. 运行测试
执行 `node test.js` 测试 Moonshine Base int8 模型的效果

### 预期优势
使用 Moonshine Base int8 模型后，预期会有以下优势：
- **更高的准确率** - Base 模型比 Tiny 模型更大，准确率更高
- **int8 量化** - 更小的模型文件，更低的内存占用，更快的推理速度
- **Moonshine 架构** - 可能比 Whisper 在某些场景下表现更好
- **VAD v4 集成** - 更精确的语音边界检测
