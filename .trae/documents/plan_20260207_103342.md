## 创建 audio-transcriber-zipformer-ctc 项目

### 任务概述
创建 `audio-transcriber-zipformer-ctc` 项目，使用 Zipformer CTC 模型进行语音识别。

### 实施步骤

#### 1. 解压模型文件
解压 `sherpa-onnx-zipformer-ctc-en-2023-10-02.tar.bz2` 文件到 `all_models` 文件夹

#### 2. 创建项目目录结构
在 `audio-transcriber-all` 文件夹下创建 `audio-transcriber-zipformer-ctc` 目录：
- `src/` - 存放源代码
- `files/` - 存放转录结果

#### 3. 复制并修改项目文件
从 `audio-transcriber-tiny` 复制所有文件到 `audio-transcriber-zipformer-ctc`：
- `src/index.ts`
- `src/transcriber.ts`
- `src/sherpa-onnx-node.d.ts`
- `package.json`
- `tsconfig.json`

#### 4. 修改 package.json
更新 `audio-transcriber-zipformer-ctc/package.json`：
- name: `audio-transcriber-zipformer-ctc`
- description: `Audio transcription module using VAD v4 + Zipformer CTC`

#### 5. 更新 TypeScript 类型定义
修改 `src/sherpa-onnx-node.d.ts`，添加 Zipformer CTC 模型配置接口：
```typescript
export interface OfflineZipformerCtcModelConfig {
  model: string;
}

export interface OfflineModelConfig {
  zipformerCtc?: OfflineZipformerCtcModelConfig;
  tokens: string;
  numThreads: number;
  provider: string;
  debug: number;
}
```

#### 6. 修改 transcriber.ts
更新 `src/transcriber.ts` 中的模型创建逻辑：
- 添加 Zipformer CTC 配置接口
- 修改 `createRecognizer()` 方法，使用 Zipformer CTC 配置结构
- 更新日志输出，显示 "Processing audio with VAD v4 + Zipformer CTC..."

#### 7. 修改 test.js
更新 `test.js` 中的模型配置：
```javascript
const config = {
  zipformerCtc: {
    model: 'D:/MyProjects/c_projects/sherpa-onnx/all_models/sherpa-onnx-zipformer-ctc-en-2023-10-02/model.int8.onnx',
  },
  vad: {
    model: 'D:/MyProjects/c_projects/sherpa-onnx/all_models/silero_vad_v4.onnx',

```

#### 8. 构建项目
运行 `npm run build` 编译 TypeScript 代码

#### 9. 运行测试
执行 `node test.js` 测试 Zipformer CTC 模型的效果

### Zipformer CTC 模型特点

1. **CTC 架构** - 使用 CTC (Connectionist Temporal Classification) 架构
2. **int8 量化** - 模型文件是 int8 量化版本
3. **高性能** - Zipformer 是一个高性能的语音识别模型
4. **简单配置** - 只需要一个 model 文件，不需要 encoder/decoder/joiner
5. **VAD v4 集成** - 更精确的语音边界检测

### 预期优势

使用 Zipformer CTC 模型后，预期会有以下优势：
- **更高的准确率** - Zipformer 是一个高性能的语音识别模型
- **int8 量化** - 更小的模型文件，更低的内存占用，更快的推理速度
- **VAD v4 集成** - 更精确的语音边界检测
- **CTC 架构** - 可能比 Transducer 架构更简单、更快速
